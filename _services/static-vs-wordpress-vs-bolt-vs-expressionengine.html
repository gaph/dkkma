Option 1. Static Website (no CMS)
Ideal for small to medium sites, low overhead

Why should I go static?

A static website usually only consists of a few pages with content that does not change very often. These sites have lower once-off cost to develop than dynamic sites, but if you need to make changes often, it could be advisable to rather go for a dynamic site, as costs are associated every time your developer needs to make changes to a static web page.

How does static compare?

PRO static

  • Fast: All files are static, no processing required.
  • Secure: No PHP, no Database, nothing to hack.
  • No overhead: No updates. Nothing that needs fixing.

CONTRA static

  • No dynamic content: The site is pre-generated so there's no dynamic content.
  • No backend: You will need a HTML developer to make changes.

Option 2. Basic CMS
Ideal for small to medium sites with easy to use backend

Why should I use a CMS?

A content management system (CMS) is a software that allows publishing, editing and modifying content without HTML knowledge. The core function and use of content management systems is to store and organize files, provide version-controlled access and enable dynamic elements.

How does it compare?

PRO CMS

  • Easy to modify: Edit your website without prior HTML knowledge.
  • Dynamic: Display dynamic content such as company stats.
  • Powerful: User accounts, e-commerce, property portal - everything is possible.

CONTRA CMS

  • Initial cost: It takes more time to develop a CMS based website.
  • Maintenance: A CMS always requires some maintenance, regular updates.

Once your website goes online, it is the editor who spends the most time working on the website. So we made the actual editing of your content as straighforward as possible.
